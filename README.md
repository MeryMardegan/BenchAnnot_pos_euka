# benchannot_pos

Post-processing and comparative analysis of functional annotation outputs generated by different annotation tools in well-annotated eukaryotic proteomes.

## Overview

This repository contains the downstream analyses used to compare the outputs of functional annotation tools applied to two well-characterized eukaryotic model organisms:

- *Saccharomyces cerevisiae*
- *Drosophila melanogaster*

The main goal is not only to assess whether proteins receive annotations, but also to evaluate the **informativeness of the assigned functional descriptions**. In this way, the repository supports a comparative benchmark of annotation performance in terms of both **coverage** and **annotation quality**.

These analyses were designed as a post-processing step after running annotation tools such as:

- **KofamScan**
- **eggNOG-mapper**
- **InterProScan**
- **PANNZER2**

---

## Scientific rationale

Functional annotation is a central step in genome and proteome interpretation. However, different annotation tools vary not only in the number of proteins they annotate, but also in the biological usefulness of the descriptions they assign.

A protein may be counted as "annotated" by a tool, yet still receive a vague or generic label such as:

- hypothetical protein
- putative protein
- unknown protein
- uncharacterized protein
- DUF-containing protein

Because of this, simple annotation counts are not sufficient to evaluate tool performance.

This repository focuses on a more informative comparison framework by asking:

- How many proteins are recovered by each tool?
- How many remain without annotation?
- How many annotations are biologically informative?
- How much information is lost relative to the reference annotation?
- How much overlap exists between tools?

Using *S. cerevisiae* and *D. melanogaster* as benchmark organisms allows comparison against well-established references, making it possible to examine not only annotation presence/absence but also annotation interpretability.

---

## Objectives

The analyses in this repository aim to:

1. Standardize reference and tool outputs for comparison  
2. Measure annotation coverage for each tool  
3. Identify proteins with no hit in each output  
4. Classify annotation descriptions as informative or non-informative  
5. Compare annotation patterns across tools  
6. Generate tables and figures for interpretation, reporting, and manuscript preparation  

---

## Benchmark design

### Reference-based comparison

The benchmark uses well-annotated reference proteomes as the basis for comparison. Protein identifiers are standardized so that the outputs of different tools can be aligned to the same protein set.

### Presence/absence of annotation

For each protein in the input set:

- **Hit**: the protein appears in the output of a tool
- **No-hit**: the protein does not appear in the output of a tool

This step evaluates annotation coverage.

### Functional description quality

Exact text matching is not used as the main criterion because different tools and databases often describe the same function using different wording.

Instead, the analyses focus on the **biological information content** of the assigned description. Descriptions are classified into two broad groups:

- **Informative**
- **Non-informative**

Descriptions containing generic terms such as:

- hypothetical
- putative
- unknown
- uncharacterized
- DUF

are considered **non-informative**, as they provide limited functional interpretation.

This strategy makes it possible to identify cases where a protein known in the reference is downgraded to a vague annotation in a tool output, representing a potential **loss of information**.

## Organisms analyzed

### *Saccharomyces cerevisiae*
Used as a compact eukaryotic model with a highly curated annotation set, suitable for benchmarking annotation recovery and description quality.

### *Drosophila melanogaster*
Used as a more complex eukaryotic model. Its annotation presents additional challenges, especially because of **isoform structure**, which must be considered carefully during post-processing and comparison.

## Repository contents

This repository is organized around Jupyter notebooks used for reference preparation, input harmonization, post-processing of tool outputs, and generation of figures and summary tables.

### Main notebooks

#### `reference.ipynb`
Prepares the *Drosophila melanogaster* reference files and generates derived tables for later comparison.

Main tasks:
- parse structural and protein reference files
- rewrite or standardize identifiers
- generate grouped and filtered tables
- prepare isoform-aware and isoform-reduced versions of the reference

Typical outputs include:
- `drosophila_melanogaster.tsv`
- `drosophila_melanogaster_separated.tsv`
- `drosophila_melanogaster_without_isoforms.tsv`
- `drosophila_melanogaster_grouped.tsv`
- `drosophila_melanogaster_grouped_no_isoforms.tsv`
- `drosophila_melanogaster_grouped_unique.tsv`

---

#### `input.ipynb`
Processes the structural input files and creates intermediate tables connecting:

- original GFF entries
- processed protein FASTA files
- reference-derived identifiers

Main tasks:
- filter GFF features
- create simplified comparison tables
- connect input protein sets to processed outputs
- prepare organism-specific tables for downstream analysis

Typical outputs include:
- `saccharomyces_cerevisiae.filtered.gff`
- `saccharomyces_cerevisiae.filtered.tsv`
- `saccharomyces_cerevisiae_gffread.tsv`
- `drosophila_melanogaster.filtered.gff`
- `drosophila_melanogaster.filtered.tsv`
- `drosophila_melanogaster_gffread.tsv`

---

#### `sc_processamento.ipynb`
Performs the post-processing of annotation outputs for *Saccharomyces cerevisiae*.

Main tasks:
- read outputs from all annotation tools
- standardize relevant columns
- identify hits and no-hits
- compare tool outputs against the reference
- classify annotations by informativeness
- generate summary tables and plots

---

#### `dm_processamento.ipynb`
Performs the post-processing of annotation outputs for *Drosophila melanogaster*.

Main tasks:
- read outputs from all annotation tools
- integrate tool results with the drosophila reference tables
- account for isoform-related structure
- classify annotations
- compare coverage and information content across tools
- generate organism-specific summaries and figures

---

#### `graph.ipynb`
Centralizes or refines figure generation for final visual outputs.

This notebook may include:
- stacked bar plots
- overlap summaries
- UpSet plots
- final formatting for report/manuscript figures

---

## Annotation tools considered

The analyses assume that annotation outputs have already been generated externally for the following tools:

- **KofamScan**
- **eggNOG-mapper**
- **InterProScan**
- **PANNZER2**

This repository does **not** run the annotation tools themselves. Instead, it focuses on the **downstream comparative analysis** of their outputs.
